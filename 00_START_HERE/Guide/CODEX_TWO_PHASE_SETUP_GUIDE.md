# Korea Grand Tour Codex Setup — Two-Phase System

The purpose of this setup is to create a working system for the Korea Grand Tour project.

The final goal is **not** to create skills. Skills are tools that come later.

## Phase 1 — Project System Setup

Use:

```text
00_START_HERE/CODEX_PROMPTS/PROMPT_01_PROJECT_SYSTEM_SETUP.md
```

Goal:

- Codex understands Korea Grand Tour deeply.
- The local folder is inspected and organized.
- The architecture and folders are created.
- Focused context files are created.
- Privacy/local-only rules are created.
- `.gitignore` and GitHub safety are prepared.
- Cross-checking and continuous improvement rules are established.
- Skill/plugin backlog is created, but skills are **not** created yet.

## Phase 2 — Skill Creation After Setup

Use only after Phase 1 is done:

```text
00_START_HERE/CODEX_PROMPTS/PROMPT_02_SKILL_CREATION_AFTER_SETUP.md
```

Goal:

- Create skills using `$skill-creator`.
- Start with foundation skills only.
- Then create immediate execution skills.
- Add eval prompts, cross-check rules, improvement rules, and output contracts.
- Plan plugins carefully.

## Which prompt should I use first?

Use Prompt 01 first.

Do not start with Prompt 02.

## Important principle

The Korea Grand Tour operating system comes first.

Skills and plugins should be created only when the folder is clean, context files exist, privacy rules are clear, and GitHub safety is prepared.
