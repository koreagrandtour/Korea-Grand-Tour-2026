# KGT Skill And Plugin Blueprint

Purpose: keep the Korea Grand Tour operating system fast, safe, and low-token.

## Skill Layers

1. Foundation: context routing, quality, project management, privacy, Git safety, asset inventory.
2. Execution: participants, copy, social, route, sponsor, website, visual deliverables.
3. Plugins: only add when a repeated task needs live access, file creation, publishing, or external-state readback.

## Plugin Rule

Do not build plugins from desire alone. Build or connect only when:

- a skill repeatedly needs live data or external writes
- privacy risk is understood
- fallback workflow is documented
- user approval is clear for live publishing, sending, or private data access

## Low-Token Pattern

- Skills route to focused context files.
- Scripts handle repeated scans/checks.
- Status files store trigger/eval matrices.
- Final answers report outcomes, not long internal checklists.
